Liquid layouts are extremely useful for web development in situations when the developer is not sure as to the specifics of the monitor size, window size or browser the web page will be viewed on. By structuring panels, labels, headers, and embedded objects based off of a percentage of the screen size, or adjusting said objects to better suit the viewing environment, it allows a developer to account for changes in the viewing environment. This can be seen on this webpage- scaling the window reduces the size of the various columns. A change in screen resolution will cause a different image to be drawn below, so as to not overfill smaller screens. It also tries multiple ways of detecting screen size so as to avoid compatibility issues with older versions of Internet Exporer.
